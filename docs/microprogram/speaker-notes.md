## 5. JOSUI：問題與產品定位｜1:40

第一個想分享的是 JOSUI。它是一個為 shadcn/ui 與 Tailwind CSS 設計的主題色彩工具，起點是我觀察到設計與開發在色彩 token 上常需要反覆對齊：即使只是調整一個主題色，也很難即時確認整體元件會怎麼變化。

因此我把這個問題做成工具。使用者可以在瀏覽器中調整 token、即時預覽元件效果，並將整理好的色彩設定帶回自己的專案。

這個專案對我來說不只是練習 React，而是從問題定義、互動設計、元件實作到 Cloudflare 部署都由我獨立完成的一次產品實驗。

## 6. JOSUI：技術實作｜1:20

技術上，我希望預覽結果是真實且即時的，而不是只做一張模擬畫面。

所以我將主題色彩的真實來源放在 CSS custom properties，透過 CSSOM 讀寫 `:root` 與 `.dark` 的 variables；React 主要負責面板開關、目前 preset 等 UI 狀態。這樣調整色彩後，shadcn/ui 元件會立即更新，使用者也能匯出 Tailwind v3 或 v4 的 token。

這段實作讓我更理解：在工具型產品中，要先釐清真正的資料來源與責任邊界，才能讓功能容易維護和延伸。

## 7. JOSUI：AI 協作｜0:55

JOSUI 也是我實際把 AI 放進開發流程的專案。AI 協助過需求與元件設計發想、React 與 TypeScript 的初稿、樣式調整，以及部署與文件整理；但我最常使用它的地方，是除錯與問題分析。

當問題牽涉到元件狀態、樣式呈現或主題 token 行為時，我會先整理最小重現情境，再請 AI 協助檢查可能原因與排查方向。最終仍會回到程式碼與畫面本身，確認型別、資料流、元件邊界與實際結果是否符合預期。

對我來說，AI 是加速分析與驗證的協作工具，不能取代工程判斷。

## 8. 都市探針 City Probe：專案概覽｜1:35

接下來是我在勝崨參與的都市探針 City Probe。它是配合大學學術研究開發的智慧城市市民參與平台，前端涵蓋登入認證、權限控制、資料表格、儀表板、地圖與多語系等功能。

我主要負責 Nuxt 3 頁面與元件開發、API 串接與狀態管理，也處理使用者資料與權限，以及表單、資料查詢和地圖相關功能。

都市探針讓我第一次更具體接觸到智慧城市的資料應用，也讓我發現，前端在真實場域裡不只是介面，而是協助使用者理解資訊、完成操作與做出決策的一環。

## 9. 都市探針 City Probe：技術挑戰｜1:35

這個專案最需要特別處理的是 Nuxt SSR 與地圖套件之間的邊界。

地圖相關套件需要瀏覽器環境才能執行，如果直接在伺服器端載入，會造成執行錯誤。因此我把地圖與瀏覽器限定套件安排在 client-only 環境，並透過 Nuxt plugins 管理表格、tooltip、日期選擇等瀏覽器端套件。

狀態管理方面，我使用 Pinia 管理使用者、語系與主題等跨頁共用資訊，並使用 TypeScript 約束 API 資料與元件 props。這讓我理解，當產品功能變複雜時，架構的目的不只是把功能拆開，而是讓後續維護與擴充仍然有清楚邊界。

## 10. FUKUKU 官網｜1:00

FUKUKU 是一個 Nuxt 3 官網專案。設計師會先和業主確認設計稿，我負責將確認後的設計轉換成可維護的前端架構、頁面元件與視覺實作。

這個專案的重點不只是畫面還原。官網需要兼顧搜尋曝光、跨裝置閱讀和載入效率，因此我使用 Nuxt 3 SSR 處理 SEO，配合 RWD 維持不同裝置上的閱讀體驗，也透過圖片 Lazy Loading 與 Code Splitting 降低初次載入負擔。

它讓我更熟悉如何在設計、品牌呈現與工程實作之間，做出合適的取捨。
